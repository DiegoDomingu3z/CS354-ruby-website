Documentation

Documentation for PHP 8: https://www.php.net/manual/en/


PHP files are naturally HTML files with PHP code embedded in them. All PHP code must be wrapped in <?php ?> or else it will be rendered as HTML.

You can omit the closing tag if the rest of the document is PHP.


PHP, being such a long standing language for running live websites, also has package managers for installing and using resources for and in PHP.

PHP Extension Community Library (PECL) (OUTDATED)
PHP Installer for Extensions (PIE)
PHP Extension and Application Repository (PEAR)
Composer (Like Rust's Cargo)
Packagist (Another repository)


Installing PHP on most systems requires a bit of effort, only Windows binaries are provided. Though most package managers will download and build PHP for you which is the point.

Unlike other languages, PHP is unique in which it's built to serve from two states.

You can execute php scripts from the command line by running php <filename>

Or by running php -r <php-code>

This does not treat all requests like it is servicing a web page, so no headers or extra information get returned.


The other way PHP is serviced is CGI, instead of being treated as a command line program, it instead assumes and treats all incoming and outgoing requests as web requests. This is the version that all websites use.
